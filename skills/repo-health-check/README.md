# repo-health-check

Run a security and health audit on a GitHub repository using [OpenSSF Scorecard](https://github.com/ossf/scorecard).

Analyzes results in context: a weekend CLI tool gets different priorities than a production service. Classifies the project into a tier (hobby, active, production) before presenting findings, so recommendations are proportionate to actual risk.

## Triggers

Use when the user mentions: repo security, branch protection, security audit, scorecard, supply chain security, repo hardening, dependency pinning, or wants to check if their GitHub project follows security best practices. Also trigger on "is my repo secure?" or "what should I fix first?" about a GitHub repo.
